## 2. โครงสร้างหน้าเว็บ (Page Structures)

### 📊 Dashboard (หน้าภาพรวม)
- **ข้อมูลที่ต้องแสดง**:
    - **Personalized Greeting**: คำทักทายต้อนรับผู้ใช้งานแบบเฉพาะเจาะจงตามชื่อโปรไฟล์
    - สรุปจำนวนแปลงและต้นไม้ทั้งหมด
    - กราฟสถานะสุขภาพ (Good/Fair/Poor)
    - รายการงานที่ต้องทำในวันนี้ (Daily Tasks)
    - สรุปสถานะการเงินล่าสุด (Income/Expense)
    - **Farm Location & Weather Forecast**: แสดงระบบระบุตำแหน่งฟาร์ม (รองรับการค้นหาพิกัดโดย OpenStreetMap API) ร่วมกับการแสดงพยากรณ์อากาศรายวันล่วงหน้า 5 วัน (เชื่อมต่อ Open-Meteo API) พร้อมแถบการแจ้งเตือนภัยเกษตรและ Skeleton Loading สำหรับความสมูทในการโหลดข้อมูล

### 🌿 Plot Management (หน้าจัดการแปลงและต้นไม้)
- **โครงสร้างหน้า**:
    - **List Side**: รายชื่อแปลงทุเรียนทั้งหมด พร้อมแสดงรายละเอียดพื้นที่และจำนวนต้นอย่างเด่นชัด
    - **Detail Side**: รายชื่อต้นทุเรียนในแปลงที่เลือก (แยกตามสุขภาพและสายพันธุ์)
- **ฟีเจอร์สำคัญ**:
    - การสลับมุมมองระหว่างแปลงต่อแปลง
    - **Delete Plot**: เพิ่มระบบลบแปลงทุเรียนพร้อมหน้าต่างย่อย (Confirmation Dialog) ยืนยันความปลอดภัยเพื่อป้องกันการกดพลาด
    - **Bulk Add Trees**: เพิ่มฟอร์มใน `TreeForm` สำหรับการเพิ่มต้นทุเรียนแบบเป็นกลุ่ม (Bulk Addition) พร้อมรันหมายเลขต้นไม้อัตโนมัติ (เช่น A-001 ถึง A-010) ในการบันทึกครั้งเดียว
    - การพิมพ์ QR Code สำหรับติดที่ต้นไม้ (ทั้งรายต้นและสั่งพิมพ์รวมเป็นชุด)
    - ระบบ Batch (รุ่นทุเรียน): หนึ่งต้นอาจมีทุเรียนหลายรุ่นพร้อมกัน

### 📝 Activity Log (หน้าบันทึกกิจกรรม)
- **ประเภทกิจกรรม**: ใส่ปุ๋ย, พ่นยา, ให้น้ำ, ตัดแต่งกิ่ง, ตรวจสอบ, เก็บเกี่ยว
- **ประวัติ**: แสดงประวัติกิจกรรมย้อนหลังพร้อมระบบฟิลเตอร์ตัวกรองตามประเภทของกิจกรรมอย่างรวดเร็ว

### 📅 Task Planner (หน้าวางแผนงาน)
- **ปฏิทิน**: เลือกวันที่เพื่อดูและเพิ่มงาน โดยมีการซิงค์ข้อมูลวันที่ที่เลือกกับรายการงานโดยอัตโนมัติ
- **สถานะงาน**: Pending, Done, Cancelled พร้อมแถบความสำคัญสูง/กลาง/ต่ำ

### 💰 Finance (หน้าการเงิน)
- **รายการ**: บันทึกรายรับและรายจ่ายโดยสามารถคลิกที่บัตรสรุป (Summary Cards) เพื่อคัดกรองประเภทรายการได้ทันที
- **วิเคราะห์**: กราฟวงกลม (Pie Chart) แสดงสัดส่วนค่าใช้จ่าย และกราฟเส้นแสดงแนวโน้มกำไรรายเดือน

### 📖 Knowledge Base (หน้าคลังความรู้)
- รายการบทความพร้อมระบบค้นหาและแยกหมวดหมู่ บทความใช้รูปภาพชนิด `.avif` เพื่อความรวดเร็วในการโหลด และแสดงรายละเอียดบทความผ่าน Pop-up Modal

### 👤 Profile & Local Authentication
- **AuthModal**: ฟอร์มล็อกอินจำลองสำหรับการเข้าถึงระดับนักพัฒนา (Development Credentials) พร้อมคำแนะนำการใช้งาน
- **ProfileModal**: บอร์ดแสดงข้อมูลผู้ใช้ที่รองรับการอัปโหลดรูปภาพโปรไฟล์ (Avatar Upload) พร้อมพรีวิวสด และความสามารถในการแก้ไขชื่อส่วนตัวและแสดงสถานะความปลอดภัยสิทธิ์ผู้ใช้

## 3. ตรรกะสำคัญ (Core Business Logic)

- **Flower Stages (ระยะดอก/ผล)**: ทุเรียนมีการเจริญเติบโตหลายระยะ (เช่น ระยะใบอ่อน, ระยะมะเขือพวง, ระยะดอกบาน, ระยะผลขยายตัว)
- **Harvest Prediction**: วันเก็บเกี่ยวจะถูกคำนวณโดยอัตโนมัติ (ประมาณ 120 วัน หลังจากวันที่ดอกบาน)
- **Health System**: การแบ่งระดับสุขภาพต้นไม้ (Good = เขียว, Fair = ส้ม, Poor = แดง)
- **Dark Mode**: ระบบสลับโทนสีหน้าสว่าง/มืด (Light/Dark Mode) เพื่อการทำงานกลางแดดจ้าและการปกป้องสายตาขณะตรวจงานในเวลากลางคืน
- **PWA (Progressive Web App)**: รองรับหน้าต่างบันทึกทางลัด (Install Prompt), ไฟล์ควบคุม `public/manifest.webmanifest` เพื่อให้ผู้ใช้งานสามารถกดบันทึกแอปลงหน้าโฮมบนมือถือเหมือนแอปพลิเคชันปกติ

## 4. ความต้องการด้านดีไซน์ (Design Requirements for AI)

- **Concept**: Modern Nature UI (โทนสีเขียวธรรมชาติ)
- **Aesthetic**: Premium, Clean, Rounded Corners (Radius: 0.75rem - 2.5rem)
- **Visibility**: ต้องมองเห็นได้ชัดเจนแม้ใช้งานกลางแดด (High Contrast) และรองรับ Dark Mode เต็มรูปแบบ
- **Responsiveness**: 
    - Desktop: 2-Column Split View (List + Detail)
    - Mobile: 1-Column List-to-Detail
- **Performance**: โหลดเร็ว ป้องกันปัญหา Layout Shift โดยใช้ Skeleton Loading ในจุดที่มีข้อมูลภายนอก
